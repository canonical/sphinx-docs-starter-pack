Release versioning
------------------

Starter pack version naming follows the Semantic Versioning 2.0.0 scheme with
modifications. It has numbers for major and minor version, but *not* patch versions.

.. list-table::
    :header-rows: 1

    * - Version
      - Example
      - Signifies
    * - Major
      - **1**.3
      - Improvements that aren't backward-compatible.
    * - Minor
      - 1.\ **3**
      - New or improved features that are simple to adopt and backward-compatible.
